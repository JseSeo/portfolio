AWS 기반 포트폴리오 사이트 설계

아키텍처 구성

주요 AWS 서비스 구성

CloudFront (CDN)

역할:
  • 정적 웹 페이지의 글로벌 전송
  • 엣지 로케이션을 통한 콘텐츠 캐싱
장점:
  • 글로벌 캐싱을 통한 속도 최적화
  • HTTPS 및 DDoS 방어 기본 제공

S3 (Simple Storage Service)

역할:
  • 정적 콘텐츠 저장 및 웹 호스팅
장점:
  • 비용 효율적인 저장소
  • 99.999999999%의 데이터 내구성
  • 간편한 정적 웹 호스팅

3. API Gateway

역할:
  • 클라이언트 요청 처리 및 Lambda 연동
  • Soft Delete 로직 처리: 삭제 요청 시 데이터를 실제로 삭제하지 않고 is_deleted 플래그를 업데이트
방명록 기능 로직 처리:
  • 데이터 작성(Create): 사용자가 입력한 방명록 내용을 DynamoDB에 저장
  • 데이터 조회(Read): is_deleted가 false인 방명록 데이터만 조회하여 사용자에게 반환
  • 데이터 수정(Update): 특정 방명록 항목의 내용을 수정
장점:
  • 확장성 있는 백엔드 통합
  • IAM 및 API Key로 보안 강화
  • 서버리스 환경 최적화로 비용 절감과 유지보수 간소화

AWS Lambda

역할:
  • 서버리스 컴퓨팅 처리
  • 데이터베이스 작업 수행
장점:
  • 사용량 기반 과금
  • 서버 관리 불필요
  • 자동 확장 지원

DynamoDB

역할:
  • 데이터베이스 계층
  • Soft Delete 지원
장점:
  • 고성능 데이터 처리
  • Lambda와의 원활한 통합
  • 확장성 있는 데이터 관리

아키텍처의 장점

  • 확장성: 자동 확장 기능을 통한 트래픽 대응
  • 비용 효율성: 서버리스 환경의 사용량 기반 과금
  • 보안: 다중 보안 계층 구현
  • 유지보수 간소화: 서버 관리 최소화
